# E17 data probe — days-to-cover / short interest: BLOCKED-ON-DATA

**2026-07-11 (CST) · M7 task 32, probe step per PRD (no prereg — the probe

## Question
Is free, point-in-time HISTORICAL short-interest (→ days-to-cover) available
for this project's **exchange-listed** universe (39 large-caps, 29 ETFs) back
far enough to cover the 2000–2013 gate window?

## Findings (probed 2026-07-11)
- **yfinance:** current snapshot only (`shortRatio`, `sharesShort`) — no
  history. (Confirmed in the 2026-07-12 data-type probe.)
- **FINRA free short-interest data** ([FINRA metadata](https://www.finra.org/finra-data/browse-catalog/equity-short-interest)):
  the free download/API dataset is the **OTC market** dataset
  (`api.finra.org/data/group/otcMarket/name/equityShortInterest`), and per
  FINRA's own file metadata, **prior to June 2021 it contains OTC securities
  only and does NOT reflect exchange-listed short interest.** Downloadable
  archives begin **2014**, and are OTC.
- **Endpoint probe:** POST to the FINRA equityShortInterest API returned
  HTTP 400 for `AAPL`/`MSFT` queries ("field not available") — the dataset is
  the OTC-market set (path `otcMarket`), consistent with the exchange-listed
  exclusion.
- **Exchange-listed short-interest history** (NYSE/Nasdaq, back to 2000s) is
  available only from **paid vendors** (e.g. QuantQuote and similar).

## Verdict: BLOCKED-ON-DATA
The Hong-Li-Ni days-to-cover anomaly requires exchange-listed short interest;
free sources cover our universe only from ~June 2021 (OTC-only before), which
cannot reach the 2000–2013 gate window. Per the PRD, E17 is recorded
**BLOCKED-ON-DATA and closed without a prereg** — no strategy was run, nothing
was fabricated or stubbed.

**Unblock conditions:** a paid short-interest history feed (a new Evan budget
decision), or restricting the test to a ≥2021 forward window (too short for
the program's regime discipline — noted, not adopted).
